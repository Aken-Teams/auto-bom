## 一、罐头档的真实结构

上传的「罐头对照表」每一行 = **一个罐头定义**，罐头类型藏在描述栏（第 11 栏）的前缀里：

| 类型 | 描述前缀 | 是否有 WAF code | 范例 |
|------|---------|----------------|------|
| 焊接罐 weld | `焊接/…` | 有 | `SMC_DW0163`（绑 `WAF108622_CP`） |
| 成型罐 mold | `成型/…` | 无（通用） | `SMC_MD0015`（Auto Mold） |
| 包装罐 pack | `包裝/…` | 无（通用） | `SMC_PA0009`（TR 7"/0.8K）、`SMC_PA0011`（TR 13"/3K） |

> 历史 bug：旧版解析「无 WAF 就跳过」，导致 21 行只读进 18 行，成型/包装的通用罐头全丢。已修复为按描述前缀分类，全部读入。

## 二、目前的匹配逻辑

### 焊接罐（weld）
- 按料号的**原件 WAF code 一对一精确匹配**罐头档里的焊接罐（`CanTemplate`，以 `waf_code` 为键）。
- 无需设定。

### 成型 / 包装罐（mold / pack）— 通用罐头
- 不绑 WAF，靠「配置罐头」页的**规则面板**指派。
- 候选罐头**从当前上传的罐头档自动带出**（不是写死）。

---

## 三、规则模型

一条规则 = **`字段` + `运算子` + `值`  →  `目标罐头`**

- **字段**：料号 `item_no` / TYPE `type_name` / FAMILY `family` / PACKAGE `package` / 原件 `component`
- **运算子**：
  | 运算子 | 含义 |
  |--------|------|
  | 全部 all | 无条件套用（兜底用，放最后） |
  | 包含 contains | 值是字段的子字符串 |
  | 等于 equals | 字段与值完全相等 |
  | 正则 regex | 用 Python `re.search(值, 字段)`，值当正则表达式，**区分大小写**，部分匹配；例 `_R[12]_`、`^GS1M`；语法错误视为不匹配 |
- **优先级**：同一类罐头，规则**由上到下逐条试，第一条命中即套用并停止**。所以顺序＝优先级，混用不同运算子互不冲突。

### 预设种子（每次进配置罐头自动带出）
- 成型：单个通用 → 一条「全部 → 该罐头」。
- 包装：若有 2 个候选 → 预填两条「料号 包含 `_R1_` → PA0009」「料号 包含 `_R2_` → PA0011」作为讨论起点（可改）。

## 四、设计决定

- **规则不存数据库**：每次进「配置罐头」都从**当前罐头档**重新带出预设；按「自动匹配」只是把面板规则套到本次任务，不持久化。
  - 原因：流程是「每次给新的底稿＋罐头档 → 筛选 → 配置 mapping」，规则可能每次不同，持久化反而难厘清。
  - 现状：`can_rules` 表与 `GET /api/can-rules` 接口骨架仍保留但**未启用**，未来若决定要「记住/复用规则」再开。
- **匹配会覆盖对应栏**：命中规则即写入；手动改过的值若再按一次自动匹配且规则命中会被覆盖。建议「先自动匹配，再手动调例外」。

## 五、待跟 USER 确认的问题

1. **包装罐判定键**：到底是看料号的 `R1`/`R2`，还是看 **TR 尺寸 / 晶圆尺寸**（7" vs 13"）？两者是否同一回事？
2. **成型罐**：是否永远只有一个通用、无条件套用到所有料号？会不会出现多个、要分规则？
3. **焊接罐匹配键可能不是 WAF（重点）**：USER 回报「焊接罐头料号识别为空」。
   - 实例：料号 `MB55_R1_00001`、原件 `WAF108425_CP`、替代结构 `ACP_68`，但罐头档里**没有 WAF108425_CP**，自动匹配为空。
   - USER 期望它匹配标黄那行 `WAF109296_CP`（**68mil**、SKY、6"），焊接罐描述 `…SKY_68MIL…`。
   - 推断：焊接罐真正的匹配键是 **mil 值（ACP_68 → 68mil）+ 可能加 function（SKY）**，而非原件 WAF code。这正是「未匹配 267」的根源（本次罐头档只有 18 个 WAF，底稿 372 行用 WAF 精确匹配大量对不上）。
   - **待确认**：焊接罐匹配键到底是「mil」单一，还是「mil + function」或「mil + 晶圆尺寸」？mil 是否一律由替代结构 `ACP_xx` 的数字取得？
4. **R1/R2 都不含的料号**：包装罐会留空，需要默认值（兜底「全部」规则）吗？
5. **规则是否需要跨任务记住 / 做成可复用的「规则集」**？（目前不存库）
6. **字段是否够用**：目前可用 料号 / TYPE / FAMILY / PACKAGE / 原件。是否需要用「晶圆尺寸」等其它字段判定？（注意：晶圆尺寸目前不在料号数据里，需另从原件摘要解析）
